Why PortView 275

A camera system built for accelerator environments

Designed for accelerator physicists and engineers who need reliable, calibrated imaging through a vacuum chamber viewport — without custom software development or camera expertise.

Versatile Viewport Imaging

Image scintillator screens, liquid metal jet targets, wire scanners, alignment fixtures, or anything else visible through a standard accelerator viewport. Any object, any beam diagnostic.

Native EPICS Integration

Full Channel Access PV set for all camera parameters, ROI control, LED illumination, calibration, and image acquisition. Compatible with Phoebus, MEDM, and any CA client.

Live Web Dashboard

Built-in MJPEG stream and browser-based control panel accessible from any machine on the lab network — no software installation required. Includes interactive click-and-drag ROI drawing.

Spatial Calibration

Per-axis mm/pixel calibration stored in configuration and published as EPICS PVs, enabling downstream tools to convert pixel coordinates to physical positions on the target plane.

Flexible Mounting

Mounts directly on a vacuum viewport flange, or operates freestanding on a tripod or periscope arm. Available for standard viewport sizes. Custom flange adapters and working-distance configurations on request.

Camera Options

Standard configuration uses the IMX708 rolling-shutter sensor (12 MP, f/1.8). A global-shutter variant based on the IMX296 with C-mount interchangeable optics is available for applications requiring motion freeze or precision lens selection.

Operation

From vacuum chamber viewport to EPICS PV

Image Acquisition

The PortView 275 mounts on or near a beamline viewport and images whatever is visible inside the vacuum chamber — scintillator screens, jet targets, wire scanners, or general chamber interiors. The Raspberry Pi CM5 captures frames via a CSI camera and publishes them as EPICS waveform PVs over standard Channel Access.

Mounting Options

The system can be bolted directly onto a viewport flange, mounted on a standard optical post or tripod, or integrated into a periscope arm for off-axis viewing. Custom flange adapters for standard CF/ISO viewport sizes are available.

Region of Interest

A configurable ROI crops the sensor before transmission, reducing PV array size and focusing on the area of interest. ROI is set interactively using the web dashboard's click-and-drag canvas, or via EPICS setpoints, and applied instantly without rebooting.

Focus & Exposure

The standard IMX708 lens is driven to a fixed diopter setpoint for consistent focus at a known working distance. The global-shutter variant uses a C-mount lens system, allowing focal length and aperture selection to match viewport geometry and target distance. Auto-exposure and manual gain/exposure are both available via PV.

Continuous or Triggered

A configurable autotrigger rate keeps the image PV refreshed for live viewing. Single-frame triggered acquisition is available for synchronized or pulsed-beam measurements.

The PortView 275 is operated entirely through EPICS — no GUI on the device is required during normal beamline operation.

Quick-Start
  1. Connect Ethernet to lab network
  2. Power via USB-C (5 V / 3 A)
  3. Edit /etc/vpcam/config.yaml
  4. sudo systemctl start vpcam vpcam-web
  5. Open http://<ip>:8080 in any browser

Global Shutter Option (IMX296)
For applications requiring freeze-frame capture of fast-moving targets — wire scanners, oscillating structures, or any subject where rolling-shutter skew is unacceptable — a variant using the Sony IMX296 global-shutter sensor (1.58 MP, 1440 × 1080, C-mount lens) is available.
